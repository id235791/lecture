원형 연결 리스트
	마지막 노드가 리스트의 첫번째 노드와 연결되어 있는 구조
	리스트의 구조를 원형으로 만든 연결 리스트
	단순 연결 리스트의 마지막 노드의 링크필드(next)에
	첫번째 노드의 주소를 저장하여 구성한다.

	장점 : 순회하여 이전 노드로 접근하는게 가능해진다.
	Cycle 형태를 이루는 구조의 구현에 용이하다.
	탐색시 불필요한 순회를 안할 수 있다.

	단점 : 구현이 어렵다. 메모리 사용이나 탐색 속도가
	여전히 떨어진다.

이중 연결 리스트
	연결이 양쪽으로 두번 되어있는 리스트
	앞의 노드, 뒤의 노드 둘 다 연결되어 있는 리스트

	장점 : 이전 노드로 이동하기가 편하기 때문에 탐색시의 속도가 굉장히 빠르다.
	단점 : 구현이 어렵다. 메모리 사용이 단일 연결 리스트보다 더 많다.

이중 연결 리스트의 Node
	data : 데이터
	next : 다음 노드의 주소값 저장
	prev : 이전 노드의 주소값 저장
