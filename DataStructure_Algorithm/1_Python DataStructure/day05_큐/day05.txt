큐
	선입선출(First In First Out) - FIFO
	삽입과 삭제가 서로 다른 입구에서 일어난다.
	한쪽에서는 삽입, 반대쪽에서는 삭제

	장점 : 시간 순서대로 처리해야 할 시 편리하다.
	단점 : 삽입과 삭제하는 방향이 정해져있다.

	front	: 삭제되는 방번호
	rear	: 추가될 방번호

선형큐
	리스트 이용, 고정 크기

원형큐
	리스트 이용, 고정 크기, 삭제시 다시 빈공간 활용 가능

연결큐
	rear변수를 새로운 데이터를 추가할 때마다 newNode로 바꿔주기
	첫 추가일때는 front도 현재 추가한 노드로 바꿔준다.
	첫 추가가 아니라면 현재 rear의 next에 newNode를 연결해준다.
	삭제시에는 front위치의 데이터를 가져오고, 현재 front가 가지고 있는
	다음 노드의 주소값을 front변수에 대입해준다.
