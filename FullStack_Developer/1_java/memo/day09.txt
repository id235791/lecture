Wrapper 클래스
	기본형 타입인 값들을 객체로 사용 가능하게 도와주는 클래스
	기본형 타입의 클래스 타입
	기본형 타입의 값을 마치 객체로 감싸고 있다고 보여서 Wrapper 클래스라 부름

	int	-->	Integer
	double	-->	Double
	char	-->	Character
		...

	1. 박싱
		Wrapper클래스명 객체명 = new Wrapper생성자(값);
		Wrapper클래스명 객체명 = (Wrapper클래스명)값;

	2. 언박싱
		wrapper객체.~~Value();

	jdk 1.5 이상부터는 오토를 지원한다

	Wrapper클래스명 객체명 = 값;		//오토박싱
	기본자료형 변수명 = wrapper객체;	//오토언박싱

제네릭(Generic)
	자료형을 선언할 때 정하지 않고 사용할 때 정해주는 기법

	<키워드>
	E : Element
	T : Type
	N : Number
	K : Key
	V : Value

	1. 제네릭 클래스
		클래스 내부에서 사용될 자료형을 지정한다.

		class 클래스명<키워드> {
			내부에서 키워드를 타입으로 사용 가능
		}

	2. 제네릭 메소드
		메소드 내부에서 사용될 자료형을 지정한다.

		<키워드> 리턴타입 메소드명(){
			내부에서 키워드를 타입으로 사용 가능
		}

	3. 제네릭 인터페이스
		인터페이스 내부에서 사용될 자료형을 지정한다.

		interface 인터페이스명<키워드>{
			내부에서 키워드를 타입으로 사용 가능
		}

컬렉션 프레임워크(Collection Framework)
	많은 데이터들을 쉽고 효과적으로 관리할 수 있는
	표준화된 방법을 제공하는 클래스 및 인터페이스들의 집합
	데이터들을 관리하기 위한 틀이 짜여져 있는 클래스, 인터페이스
	자료구조의 일종

			Collection(interface)

		List(interface)		Set(interface)

구현 클래스
		ArrayList(class)	HashSet(class)
		Vector(class)		TreeSet(class)
		LinkedList(class)	...
		...

ArrayList
	Array는 배열 이라는 뜻이고 List는 요소들의 나열된 묶음이다.
	ArrayList는 배열을 이용해서 요소들을 나열해 묶어놓은 형태의 클래스이다.
	컬렉션 프레임워크중 가장 많이 사용되는 클래스이다.

	인덱스를 이용해서 내부 배열의 요소에 빠르게 접근 가능하지만
	크기를 변경시키기 위해서(요소를 추가, 삭제하기 위해서) 새로운 배열을 생성하고
	기존의 값들을 옮기는 과정을 필요로 하기 때문에 느리다.

	배열은 처음에 몇 칸을 할당할지 고정해야 했지만,
	ArrayList 객체는 값을 넣는 만큼 자동으로 늘어난다.

HashSet
	Hash는 해시값, Set은 집합이라는 뜻이다.
	집합은 중복되는 요소(원소, 값)를 포함할 수 없다.
	저장된 요소들에는 인덱스가 없기 때문에 저장 순서가 고정되어 있지 않다.
	값의 포함 여부, 값의 유무 검사 목적이 있다.

	Set은 검색의 목적이 있기 때문에 순서 정보를 관리할 필요가 없다.
