인터넷
	전 세계에 걸쳐 원거리 접속이나 파일 전송, 전자 메일 등
	데이터 통신 서비스를 받을 수 있는 컴퓨터 네트워크 시스템

클라이언트와 서버
	클라이언트(정보를 제공받는 사용자 또는 컴퓨터)
		요청↓	↑응답
	서버(정보를 제공하는 사용자 또는 컴퓨터)

프로토콜(Protocol)
	넷상에서 컴퓨터끼리 소통할 때 사용해야 하는 규칙들을
	프로토콜이라 한다. 통신하기 위해 필요한 규약

IP(Internet Protocol)
	네트워크 상에서 다른 컴퓨터와 구별될 수 있도록 고유번호(주소)를
	부여하는 프로토콜 000~255 * 4(IPv4)

http(HyperText Transfer Protocol)
	클라이언트와 서버간의 웹 페이지 등의 자원을 통신하는 규약

HyperText
	한 문서에서 다른 문서로 접근하는 개념
	문서에 주소(URL)를 붙이고 그 주소를 통해 이동, 요청 하는것을
	HyperText라 부른다.

웹 브라우저
	사용자의 요청에 맞는 주소로 찾아가서 인터넷의 컨텐츠
	(문서와 그림, 멀티미디어 파일 등)를 검색 및 열람 후
	사용자에게 보여주기 위한 응용 프로그램의 총칭

도메인(Domain)
	IP주소는 기억하고 이해하기 힘들기 때문에 이를 위해서
	이름을 부여할 수 있도록 하는 서비스
---------------------------------------------------------------
HTML(HyperText Markup Language)
	텍스트로 페이지의 구조와 데이터들을 표기하는 언어
	HTML 문서는 확장자가 .html
	HTML 문서는 문서를 작성할 수 있는 프로그램이라면 어떠한것이든지 작성이 가능하다.
	HTML 문서는 대소문자를 구별하지 않는다.
	HTML 문서는 띄어쓰기, 줄바꿈을 구별하지 않는다.
	HTML 문서는 태그 형태로 꺽쇠괄호(<명령어>)를 사용하여 표현한다.

HTML의 요소(태그)
	①<p>③웹과의 첫 만남~!②</p>

	① 여는 태그(Opening Tag)	: 요소의 이름(p)이 열고 닫는 꺽쇠 괄호 안에 있음
	② 닫는 태그(Closing Tag)	: 여는태그와 똑같이 생겼지만, 요소의 이름 앞에 슬래시(/)가 있음
	③ 내용(Content)		: 요소의 내용이며 단순한 텍스트 혹은 또 다른 태그

웹 접근성
	정상적인 웹 컨텐츠 사용이 가능한 일반 사용자부터
	고령자, 장애인 같은 신체적, 환경적 조건에 제한이 있는 사용자들을
	포함해 모든 사용자들이 동일한 조건에서 접근할 수 있도록
	웹 컨텐츠를 제작하는 기법

웹 표준
	w3c에서 제정
	여러 브라우저에서 동일한 사용자 경험을 줄 수 있도록 하는
	기술, 방법 등을 의미 / 크로스 브라우징

서식 태그
	<서식태그>내용</서식태그>

	<strong>
		텍스트를 굵게 표현하고 싶을 때 사용
		<b>태그와 동일한 기능
		웹 접근성에 사용
	<em>
		텍스트를 이탤릭체로 표현하고 싶을 때 사용
		<i>태그와 동일한 기능
		웹 접근성에 사용

	<mark>
		텍스트에 하이라이팅(형광) 효과를 적용

	<del>
		텍스트 중앙에 가로줄을 만들어 텍스트를 지운것과 같은 효과를 줌

	<ins>
		텍스트 밑에 선을 그어 텍스트에 강조효과를 줌

	<sup>, <sub>
		위첨자, 아래첨자

HTML 요소의 속성
	<태그명 속성명="속성값" 속성명="속성값" ...>내용</태그명>
	속성은 해당 요소에 대한 추가적인 정보를 제공
	속성에 대한 값은 따옴표로 꼭 묶어주지 않아도 되지만
	속성값에 띄어쓰기가 있는 경우 문제가 발생할 수 있으므로
	따옴표를 쓰기 권장함(쌍따옴표, 따옴표 둘 다 가능)

목록 태그
	블록 태그

	<ul>
		순서가 없는 목록 태그

		<ul>
			<li>내용1</li>
			<li>내용2</li>
			...
		</ul>
	<ol>
		순서가 있는 목록 태그

		<ol>
			<li>내용1</li>
			<li>내용2</li>
			...
		</ol>

		type
			1 : 숫자(1,2,...)
			A : 알파벳 대문자(A,B,...)
			a : 알파벳 소문자(a,b,...)
			I : 로마숫자 대문자(I,II,...)
			i : 로마숫자 소문자(i,ii,...)
		start
			목록의 시작 번호

	<dl>
		사용자 정의 목록 태그
		목록의 제목은 <dt>로, 목록의 내용은 <dd>로 만들음

		<dl>
			<dt>정의 제목1</dt>
			<dd>정의 제목1에 대한 내용1</dd>
			<dd>정의 제목1에 대한 내용2</dd>
			...
			<dt>정의 제목2</dt>
			<dd>정의 제목2에 대한 내용1</dd>
			<dd>정의 제목2에 대한 내용2</dd>
			...
		</dl>

HTML5의 문서 구조
	<!DOCTYPE html>
		HTML 문서의 선언 / HTML 버전 5를 나타냄

	<html lang="en">
		lang 속성은 웹 접근성에 관한 명시
		스크린 리더기에서 언어를 인식하여 자동으로 음성을 변환하거나,
		해당 언어에 적합한 발음을 제공(한국어 설정은 lang="ko")

	<meta>태그
		<head> 태그에 정보를 추가하기 위한 태그

		charset="UTF-8"
			charset 속성은 문자를 인코딩할 때 문자코드를 설정하는 역할
			우리나라는 윈도우 기본 문자셋인 EUC-KR을 사용해도 되지만,
			다국어로 인코딩 하기 위해서는 UTF-8 문자셋을 사용하기를 권장
			EUC-KR은 완성형, UTF-8은 조합형

		name="viewport" content="width=device-width, initial-scale=1.0"
			viewport : 가상의 화면
			모바일로 최적화된 사이트에 포함하는 META 속성 설정

비트맵과 벡터 이미지
	비트맵 이미지
		픽셀이 모여서 만들어진 정보의 집합
		레스터 이미지 라고 부름
		픽셀 단위로 화면에 렌더링함
		그림판, 포토샵 등 툴로 편집

	벡터 이미지
		수학적 정보의 형태들이 만들어내는 결과물
		이미지가 가지고 있는 점, 선, 면의 위치, 색상정보, 크기 등을 가지고 있음
		확대 및 축소를 해도 이미지가 깨지지 않음
		일러스트 같은 툴로 편집

이미지 파일의 형식
	jpg(jpeg)
		압축률이 뛰어나 사진이나 예술분야에 많이 사용
		가장 널리 쓰이는 이미지 포맷
		표현 색상(24비트, 약 1600만 색상)이 뛰어나 고해상도 표시장치에 적합
		핸드폰, 카메라 사진
		손실 압축

	gif
		파일 내에 다른 이미지 및 문자열 같은 정보를 저장할 수 있는 파일
		여러장의 이미지를 한 개의 파일에 저장할 수 있음(움짤, 애니메이션)
		8비트 컬러(256색상)만 지원
		비손실 압축

	png
		gif의 대체 포맷으로 개발
		8비트, 24비트 컬러 이미지 처리
		w3c 권장 포맷
		알파 채널 지원(배경을 투명하게 설정)
		아이콘

	webp
		jpg, png, gif를 모두 대체할 수 있는 구글이 개발한 이미지 포맷
		완벽한 포맷
		gif같은 애니메이션 지원
		알파 채널 지원
		손실, 비손실 압축
		
		웹 브라우저 호환성 확인 : https://caniuse.com
		무료 png 이미지 사이트 : https://www.iconfinder.com
